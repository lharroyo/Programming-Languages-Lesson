Introducción a las APIs

Aprende qué son las APIs, cómo funcionan y por qué son la base de la comunicación entre aplicaciones modernas.

1. ¿Qué es una API?

API significa Application Programming Interface (Interfaz de Programación de Aplicaciones). Es un conjunto de reglas y definiciones que permite que dos aplicaciones se comuniquen entre sí.

🍽️ Analogía: El mesero del restaurante

👤 ➡️ 🧑‍🍳 ➡️ 🍕 ➡️ 👤

Imagina que estás en un restaurante. Tú (el cliente) no puedes ir directamente a la cocina. El mesero (API) toma tu pedido, lo lleva a la cocina (servidor), y te trae la comida (respuesta).

📱

Cliente (Frontend)

La aplicación que hace la solicitud. Puede ser una app móvil, un sitio web o cualquier programa.

📡

API (Intermediario)

Define las reglas de comunicación: qué puedes pedir, cómo pedirlo y qué formato tendrá la respuesta.

🖥️

Servidor (Backend)

Procesa la solicitud, accede a la base de datos si es necesario y genera la respuesta.

💡 ¿Qué es JSON?

JSON (JavaScript Object Notation) es el formato más común para intercambiar datos en APIs. Es un texto legible que usa pares clave: valor.

3. Métodos HTTP (Verbos)

Los métodos HTTP indican qué acción quieres realizar sobre un recurso. Los 4 principales forman el acrónimo CRUD:

GET

Leer — Obtener datos del servidor. No modifica nada.

GET /api/productos
POST

Crear — Enviar datos nuevos al servidor.

POST /api/productos
PUT

Actualizar — Modificar un recurso existente completo.

PUT /api/productos/5
DELETE

Eliminar — Borrar un recurso del servidor.

DELETE /api/productos/5

📝 CRUD = Create, Read, Update, Delete

POST → Create  |  GET → Read  |  PUT → Update  |  DELETE → Delete

4. Códigos de Estado HTTP

Cada respuesta del servidor incluye un código numérico que indica el resultado de la solicitud:

200
OK — Todo salió bien
201
Created — Recurso creado
301
Moved — Recurso movido
400
Bad Request — Error del cliente
401
Unauthorized — Sin autenticación
404
Not Found — No existe
500
Server Error — Error interno
503
Unavailable — Servicio caído
💡 Regla rápida

2xx = Éxito ✅  |  3xx = Redirección 🔄  |  4xx = Error del cliente ❌  |  5xx = Error del servidor 💥

5. Tipos de APIs

🌐

REST (RESTful)

El estándar más popular. Usa HTTP, URLs como recursos y JSON como formato. Simple y escalable.

📊

GraphQL

Creado por Facebook. El cliente decide exactamente qué datos necesita. Una sola URL para todo.

📨

SOAP

Protocolo más antiguo y estricto. Usa XML. Común en bancos y sistemas empresariales legacy.

⚡

WebSocket

Comunicación bidireccional en tiempo real. Ideal para chats, juegos y notificaciones en vivo.

⚠️ En esta clase nos enfocaremos en REST

REST es el tipo de API más usado en la industria actual. Es el mejor punto de partida para entender cómo funcionan las APIs.

6. Ejemplo Práctico: Consumir una API

Veamos cómo consumir una API pública real usando JavaScript (fetch) y luego con Python (requests).

Con JavaScript (en el navegador)

Consumir API con fetch() JavaScript 
// Obtener datos de una API pública de usuarios
async function obtenerUsuario() {
    const url = "https://jsonplaceholder.typicode.com/users/1";

    try {
        const response = await fetch(url);

        // Verificar que la respuesta sea exitosa
        if (!response.ok) {
            throw new Error(`Error: ${response.status}`);
        }

        const datos = await response.json();
        console.log("Nombre:", datos.name);
        console.log("Email:", datos.email);
        console.log("Ciudad:", datos.address.city);
    } catch (error) {
        console.error("Falló la solicitud:", error);
    }
}

obtenerUsuario();

Con Python

Consumir API con requests Python 
import requests

# Obtener datos de la misma API pública
url = "https://jsonplaceholder.typicode.com/users/1"
response = requests.get(url)

if response.status_code == 200:
    datos = response.json()
    print(f"Nombre: {datos['name']}")
    print(f"Email:  {datos['email']}")
    print(f"Ciudad: {datos['address']['city']}")
else:
    print(f"Error: {response.status_code}")
